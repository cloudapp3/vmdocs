# Prepare for v0.2

::: warning Unreleased
The v0.2 code and release draft are on `master`, but there is no `v0.2.0` tag yet. The latest tagged release is `v0.1.0`. Do not treat this page as a published compatibility guarantee until the tag exists.
:::

## Result shape corrections

The planned release returns slices for Telegram batch methods:

- `CopyMessages`: `[]MessageId`
- `ForwardMessages`: `[]MessageId`
- `SendMediaGroup`: `[]Message`
- `GetUserPersonalChatMessages`: `[]Message`

Methods that can return a chat `Message` or inline `True` use `MessageOrBool` and its `AsMessage` and `AsBool` helpers.

## Upload-capable fields

Generated `Input*` fields that accept `attach://` uploads use `InputFile`. Replace plain strings with:

- `FileFromID`
- `FileFromURL`
- `FileFromPath`
- `FileFromReader`

Multipart bodies stream instead of buffering complete files.

## Polling behavior

- The root `Updates()` stream becomes opt-in.
- A poller with no sink waits instead of calling `getUpdates`.
- Registering a subscription wakes a running poller.
- `Unsubscribe()` closes the subscription channel before returning.
- Non-blocking dispatch reports dropped updates through `UpdateDispatchError`.

## HTTP deadlines

The planned defaults are 30 seconds for ordinary calls, Telegram's long-poll timeout plus 5 seconds for `getUpdates`, and 10 minutes for multipart uploads. Caller deadlines and custom HTTP client policy remain authoritative.

## Bot API 10.2

Generated definitions are refreshed from the July 14, 2026 official documentation snapshot. Update helpers include the newer `guest_message`, `managed_bot`, and `subscription` fields.

Review the source repository's [release draft](https://github.com/cloudapp3/tgbot/blob/master/docs/releases/v0.2.0.md) together with the final tagged diff before upgrading.

Then audit the affected [methods and result types](/sdk/methods-and-types), [file uploads](/sdk/file-uploads), [long-polling consumers](/updates/long-polling), and [client deadlines](/sdk/client).
