# 准备迁移到 v0.2

::: warning 尚未发布
`v0.2.0` 代码已经进入 master，但还没有对应 release tag。当前最新正式版本仍是 `v0.1.0`。tag 发布前，本页不是正式兼容性保证。
:::

## 返回形态修正

计划发布版本会为 Telegram 批量方法返回 slice：

- `CopyMessages`：`[]MessageId`
- `ForwardMessages`：`[]MessageId`
- `SendMediaGroup`：`[]Message`
- `GetUserPersonalChatMessages`：`[]Message`

可能返回聊天 `Message` 或内联 `True` 的方法改用 `MessageOrBool`，通过 `AsMessage` 和 `AsBool` 读取具体分支。

## 可上传字段

接受 `attach://` 上传的生成 `Input*` 字段改用 `InputFile`。原字符串应替换为：

- `FileFromID`
- `FileFromURL`
- `FileFromPath`
- `FileFromReader`

Multipart body 会流式发送，不再缓冲完整文件。

## Poller 行为

- 根 `Updates()` 流改为按需启用。
- 没有 sink 时等待，不调用 `getUpdates`。
- 注册订阅会唤醒已运行的 Poller。
- `Unsubscribe()` 返回前关闭订阅 channel。
- 非阻塞分发通过 `UpdateDispatchError` 报告丢弃。

## HTTP deadline

计划默认值是：普通调用 30 秒，`getUpdates` 使用 Telegram 长轮询 timeout 加 5 秒，multipart 上传 10 分钟。调用方已有 deadline 和自定义 HTTP client 策略优先。

## Bot API 10.2

生成定义已经刷新到 2026 年 7 月 14 日官方文档快照。Update 辅助方法覆盖新增的 `guest_message`、`managed_bot` 和 `subscription` 字段。

升级前应同时阅读源码仓库中的 [release draft](https://github.com/cloudapp3/tgbot/blob/master/docs/releases/v0.2.0.md)和最终 tag diff。

随后逐项检查受影响的[方法与返回类型](/zh/sdk/methods-and-types)、[文件上传](/zh/sdk/file-uploads)、[长轮询消费者](/zh/updates/long-polling)和[客户端 deadline](/zh/sdk/client)。
