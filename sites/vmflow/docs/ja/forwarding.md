# 転送エンジン

`engine/` パッケージが純粋な転送コアです。リスナー、プロキシループ、セッション、カウンタ、ルールライフサイクルだけを所有し、それ以外は持ちません。データベース、コントロールプレーン、UI のいずれにも依存しません。これが組み込み可能性を保っています。

## プロトコル

各ルールは `protocol` フィールドでプロトコルを 1 つ選びます。

| プロトコル | 挙動 |
| --- | --- |
| `tcp` | TCP ポート転送。 |
| `udp` | UDP ポート転送。 |
| `tcp+udp` | 1 つのルールから同じポートで TCP と UDP を両方転送します。 |
| `http` | HTTP フォワードプロキシ（プレーン + `CONNECT`）。**現在のビルドでは無効**。 |
| `https` | ドメインごとの証明書を使う TLS 終端フォワード。**現在のビルドでは無効**。 |

`http` と `https` はソース（`engine/https.go`、`engine/proxy.go`）および ACME/certstore パッケージとともに存在しますが、現在のビルドでは検証時に拒否され、証明書管理は結線されていません。これらは後で再度有効化するために予約されています。

## ルールごとの制御

各ルールはプロトコルに依存しないランタイム制御を持ちます：

- **`speed_limit`** — 接続ごとの双方向レート制限（トークンバケット）、単位はバイト/秒。`0` は無制限を意味します。これは意図的にシンプルでセッション単位の仕組みであり、共有されたグローバル帯域バケットではありません。
- **`max_conn`** — 同時接続数のキャップ。キャップに達すると、新規接続は即座に切断されます。`0` は無制限を意味します。
- **`idle_timeout`** — 接続ごとのアイドルタイムアウト（秒）。`0` はデフォルト（5 分 / 300 秒）を使うことを意味します。変更するとルールは再起動します。
- **`enabled`** — 無効化されたルールは設定（およびスナップショット）には残りますが起動しません。

## トラフィックカウンタ

実行中の各ルールはリアルタイムのカウンタを報告します：

- 現在の接続数
- アップロードバイト合計
- ダウンロードバイト合計

カウンターはメモリ上に保持されます。UDP の接続数はセッションの近似値です。`vmflow ctl stats` または TUI で確認してください。

## ルールの同値性

ルールはランタイムフィールド（プロトコル、アドレス、ポート、制限、`idle_timeout`）とメタデータ（`remark`、`revision`、タイムスタンプ）の両方を持ちます。エンジンは新しいスナップショットを稼働状態と比較する際にランタイムフィールドだけを比較するため、表面的な編集が無用な再起動を引き起こすことはありません — ただし `idle_timeout` の変更は再起動を引き起こします。[ルールとライフサイクル](./rules)を参照してください。
