# 포워딩 엔진

`engine/` 패키지는 순수 포워딩 코어입니다. 리스너, 프록시 루프, 세션, 카운터, 규칙 라이프사이클만 담당하며 그 외에는 아무것도 없습니다. 데이터베이스, 컨트롤 플레인, UI에 의존하지 않습니다. 덕분에 임베드하기 쉽습니다.

## 프로토콜

각 규칙은 `protocol` 필드로 프로토콜을 하나 선택합니다.

| 프로토콜 | 동작 |
| --- | --- |
| `tcp` | TCP 포트 포워딩. |
| `udp` | UDP 포트 포워딩. |
| `tcp+udp` | 단일 규칙으로 같은 포트에서 TCP와 UDP를 모두 포워딩. |
| `http` | HTTP 포워드 프록시(평문 + `CONNECT`). **현재 빌드에서 비활성화됨.** |
| `https` | 도메인별 인증서로 TLS를 종료하는 포워딩. **현재 빌드에서 비활성화됨.** |

`http`와 `https`는 소스(`engine/https.go`, `engine/proxy.go`)에 ACME/certstore 패키지와 함께 존재하지만, 현재 빌드는 검증 단계에서 이들을 거부하고 인증서 관리를 연결하지 않습니다. 나중에 다시 활성화하기 위해 예약되어 있습니다.

## 규칙별 제어

각 규칙은 프로토콜과 무관한 런타임 제어를 가집니다:

- **`speed_limit`** — 연결당 양방향 속도 제한(토큰 버킷), 단위는 바이트/초. `0`은 무제한입니다. 의도적으로 단순하고 세션별로 적용되며, 공유되는 전역 대역폭 버킷이 아닙니다.
- **`max_conn`** — 동시 연결 상한. 상한에 도달하면 새 연결이 즉시 닫힙니다. `0`은 무제한입니다.
- **`idle_timeout`** — 연결당 유휴 타임아웃(초). `0`은 기본값(5분 / 300초) 사용을 의미합니다. 변경하면 규칙이 재시작됩니다.
- **`enabled`** — 비활성화된 규칙은 설정(및 스냅샷)에는 유지되지만 시작되지 않습니다.

## 트래픽 카운터

실행 중인 모든 규칙은 실시간 카운터를 보고합니다:

- 현재 연결 수
- 업로드 바이트 총합
- 다운로드 바이트 총합

카운터는 메모리에 유지됩니다. UDP 연결 수는 세션 수의 근사치입니다. `vmflow ctl stats` 또는 TUI에서 확인하세요.

## 규칙 동등성

규칙은 런타임 필드(프로토콜, 주소, 포트, 제한, `idle_timeout`)와 메타데이터(`remark`, `revision`, 타임스탬프)를 모두 가집니다. 엔진은 새 스냅샷과 실행 중인 상태를 diff할 때 런타임 필드만 비교하므로, 단순한 편집은 불필요한 재시작을 유발하지 않습니다 — 단, `idle_timeout`을 변경하면 재시작이 발생합니다. [규칙과 라이프사이클](./rules)을 참고하세요.
