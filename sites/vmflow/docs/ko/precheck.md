# 사전 검사

사전 검사는 설정을 **적용하지 않고** 검증합니다. 모든 `reload` 전에 자동으로 실행되며, 필요할 때 직접 실행해 설정 변경을 안전하게 점검할 수 있습니다.

## 실행하기

```bash
vmflow ctl precheck
```

`vmflow ctl precheck`로 현재 설정을 검증합니다.

## 검사 항목

사전 검사는 발견 항목 목록을 생성하며, 각 항목은 **오류** 또는 **경고**입니다. 오류는 리로드를 차단하고, 경고는 차단하지 않습니다.

| 검사 | 심각도 | 비고 |
| --- | --- | --- |
| 규칙 모델 검증 | 오류 | 잘못된 규칙 필드. |
| 소스 IP 정책 | 오류 | 잘못된 모드, 호스트 이름, 잘못되거나 빈 항목, 비어 있는 활성 목록 또는 256개를 초과하는 항목. |
| `source_ip_duplicate` | 경고 | 이전 항목과 동일한 주소 또는 CIDR로 해석되는 항목. |
| `source_ip_redundant` | 경고 | 같은 목록의 더 넓은 CIDR에 이미 포함되는 항목. |
| `duplicate_rule_id` | 오류 | 같은 ID가 스냅샷에 두 번 이상 나타남. |
| 리스너 충돌 | 오류 | 두 규칙이 같은 `listen_addr:port`를 사용함. |
| 포트 바인딩 가능성 | 오류 | 리슨 포트를 실제로 바인딩해 사용 가능한지 확인함. |
| 대상 DNS 해석 | 오류 | `target_addr`이 해석되어야 함. |
| HTTPS 도메인 설정 | — | _현재 빌드에서 비활성화됨_(http/https 프로토콜이 거부됨). |
| ACME HTTP-01 주소 | — | _현재 빌드에서 비활성화됨_(ACME 하위 시스템이 꺼져 있음). |
| 낮은 포트 권한 | 경고 | 권한 포트(<1024) 바인딩은 일반적으로 상승된 권한이 필요함. |

## 응답 예시

```json
{
  "config_path": "./examples/config.yaml",
  "rule_count": 1,
  "result": {
    "ok": true,
    "error_count": 0,
    "warning_count": 0,
    "checked_rules": 1,
    "checked_time_ms": 1,
    "items": []
  }
}
```

## 리로드에서의 활용

`vmflow ctl reload`는 먼저 동일한 검사를 실행합니다. `error_count > 0`이면 리로드가 거부되며, 실행 중인 규칙은 그대로 유지됩니다. 따라서 설정 편집을 자동화로 안전하게 전달할 수 있습니다. 잘못된 설정이 부분적으로 적용되는 일은 없습니다.

::: warning 로컬 포트
포트 바인딩 가능성 검사가 잠시 로컬 리스너를 엽니다. 환경에서 로컬 소켓 생성을 차단하면, 사전 검사(그리고 리로드)가 바인딩 가능성을 완전히 검증할 수 없습니다.
:::
