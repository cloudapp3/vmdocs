# 규칙과 라이프사이클

*규칙(rule)*은 vmflow가 관리하는 단위입니다. 제한과 메타데이터를 가진 하나의 `{protocol, listen, target}` 포워딩 항목입니다. 규칙은 한 번에 하나씩, 또는 전체 desired-state 스냅샷으로 구동할 수 있는 라이프사이클을 가집니다.

## 안정적인 식별자

각 규칙은 설정/스냅샷 내에서 고유해야 하는 `rule_id`를 가집니다. 이 ID로 vmflow는 들어오는 규칙을 실행 중인 규칙과 매칭하여 무엇이 변경되었는지 판단합니다. 리로드 간에 ID를 안정적으로 유지하세요 — 그래야 diff가 동작합니다.

## 단일 규칙 작업

| 작업 | 효과 |
| --- | --- |
| `StartRule` | 규칙 하나를 시작합니다(이미 실행 중이면 안 됨). |
| `RestartRule` | 실행 중인 규칙을 중지하고 다시 시작하여 새 필드를 반영합니다. |
| `StopRule` | 실행 중인 규칙을 중지합니다. 설정은 유지합니다. |
| `RemoveRule` | 규칙을 중지하고 실행 집합에서 제거합니다. |

이 작업들은 국소적인 로컬 작업에 적합합니다. 전체 desired state가 있다면 스냅샷 적용을 권장합니다.

## 스냅샷 적용

`ApplySnapshot(rules, opts)`은 규칙의 전체 desired 집합을 받아 실행 중인 것과 조정(reconcile)합니다. `ReplaceAll`을 사용하면 스냅샷에 없는 실행 중인 규칙은 모두 중지됩니다.

각 규칙에 대해 적용은 하나의 액션을 만들어냅니다:

| 액션 | 의미 |
| --- | --- |
| `started` | 규칙이 스냅샷에 있지만 실행 중이 아니었습니다. |
| `restarted` | 규칙이 실행 중이었지만 런타임 필드가 변경되었습니다. |
| `stopped` | 규칙이 실행 중이었지만 스냅샷에 없습니다(`ReplaceAll` 사용 시). |
| `removed` | 규칙이 중지되고 제거되었습니다. |
| `unchanged` | 규칙이 실행 중이었고 런타임 필드가 변경되지 않았습니다. |
| `failed` | 규칙을 시작할 수 없었습니다(예: 포트 사용 불가). |

…그리고 요약: `Applied`, `Stopped`, `Failed`, `Total`.

::: tip "변경됨"의 기준은?
런타임 필드(프로토콜, 리슨 주소/포트, 타겟 주소/포트, 속도 제한, 최대 연결 수, `idle_timeout`, enabled)만 비교됩니다. `remark`나 타임스탬프를 편집해도 규칙이 **재시작되지 않습니다**.
:::

## 리로드

`vmflow ctl reload`는 설정 파일을 다시 읽고 `ReplaceAll = true`로 `ApplySnapshot`을 실행합니다. 먼저 [사전 검사](./precheck)를 실행하며, 오류가 있으면 리로드가 거부되고 실행 중인 규칙은 그대로 유지됩니다.

## 상태 쿼리

| 메서드 | 반환 |
| --- | --- |
| `RunningRules` / `RunningCount` | 현재 실행 중인 규칙 / 개수. |
| `Snapshot(id)` | 단일 규칙의 실시간 상태. |
| `SnapshotAll()` | 실행 중인 모든 규칙의 실시간 상태. |
| `StopAll()` | 모두 중지(예: 종료 시). |
