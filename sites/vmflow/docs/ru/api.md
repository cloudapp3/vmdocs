# Локальное управление

Поддерживаемые интерфейсы управления: `vmflow ctl`, `vmflow tui` и MCP-адаптер
`vmflow mcp` только для чтения. Демон использует внутренний транспорт, доступный
только через loopback. Этот транспорт не является публичным API интеграции, и
совместимость для внешних клиентов не гарантируется.

Управление всегда привязано к `127.0.0.1`; в конфигурации задаётся только порт:

```yaml
control_port: 19090
```

Для состояния, правил, статистики, проверки и перезагрузки используйте
[`vmflow ctl`](./ctl) и [`vmflow tui`](./tui). Для локальной диагностики с
помощью ИИ используйте [`vmflow mcp`](./mcp).

## Удалённое управление

Пробросьте loopback-порт через SSH и используйте CLI/TUI локально:

```bash
ssh -N -L 19090:127.0.0.1:19090 user@server
vmflow ctl rules
```
