# Справочник команд

vmflow — это одиночный бинарный файл с семью подкомандами. Алиасы приведены в таблице ниже.

| Команда | Алиас | Назначение |
| --- | --- | --- |
| [`daemon`](./daemon) | `d` | Запуск демона перенаправления. |
| [`ctl`](./ctl) | `c` | Опрос и управление работающим демоном. |
| [`tui`](./tui) | `t` | Терминальная панель. |
| [`version`](./version) | `v` | Вывод метаданных сборки. |
| [`update`](./update) | `u` | Проверка или установка нового релиза. |
| [`service`](./service) | `svc` | Регистрация как нативной службы ОС (старт при загрузке). |
| [`uninstall`](./uninstall) | `remove`, `rm` | Удаление одной командой с очисткой. |

## Общие клиентские флаги {#common-client-flags}

`ctl` и `tui` — клиенты [control API](./api) и используют общие флаги:

| Флаг | Переменная окружения | По умолчанию | Описание |
| --- | --- | --- | --- |
| `-addr` | _(нет)_ | `http://127.0.0.1:19090` | Базовый URL control API. |
| `-token` | `VMFLOW_CONTROL_TOKEN` | _(нет)_ | Bearer-токен при включённой аутентификации. |
| `-tls-ca-file` | `VMFLOW_TLS_CA_FILE` | _(нет)_ | Набор CA для проверки сертификата сервера control API (частные/самоподписанные CA). |
| `-tls-client-cert` | `VMFLOW_TLS_CLIENT_CERT` | _(нет)_ | Клиентский сертификат для mTLS (обязателен, когда на сервере задан `control_tls.client_ca_file`). |
| `-tls-client-key` | `VMFLOW_TLS_CLIENT_KEY` | _(нет)_ | Клиентский ключ для mTLS (используется вместе с `-tls-client-cert`). |
| `-tls-skip-verify` | `VMFLOW_TLS_INSECURE` (`1`/`true`) | `false` | Пропустить проверку сертификата сервера (опасно, только для отладки). |
| `-H` / `--header` | `VMFLOW_HEADERS` (через `;`) | _(нет)_ | Дополнительный заголовок запроса в виде `Name: Value` (можно повторять). |

## Примечания

- Прежние раздельные бинарные файлы `relayd`, `relayctl` и `relaytui` всё ещё собираются для совместимости — это тонкие обёртки над теми же пакетами, читающие ту же переменную окружения `VMFLOW_CONTROL_TOKEN` — но в релизах предпочтение отдаётся единому бинарному файлу `vmflow`.
- Туннельные команды (`tunnel-server`, `tunnel-client`, `tunnel-ctl`) и команды сертификатов (`certs`, `certs-obtain`, `certs-review`) **не включены** в текущую сборку.
