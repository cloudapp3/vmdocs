# Справочник команд

vmflow — одиночный бинарный файл с режимом foreground и семью подкомандами.

| Команда | Алиас | Назначение |
| --- | --- | --- |
| `vmflow` | - | Запуск forwarding runtime в foreground. |
| [`ctl`](./ctl) | `c` | Опрос и управление работающим демоном. |
| [`tui`](./tui) | `t` | Терминальная панель. |
| [`mcp`](./mcp) | - | MCP-сервер stdio только для чтения для локального работающего демона. |
| [`version`](./version) | `v` | Вывод метаданных сборки. |
| [`update`](./update) | `u` | Проверка или установка нового релиза. |
| [`service`](./service) | `svc` | Регистрация как нативной службы ОС (старт при загрузке). |
| [`uninstall`](./uninstall) | `remove`, `rm` | Удаление одной командой с очисткой. |

## Общие параметры управления {#common-client-flags}

Встроенные `ctl`, `tui` и `mcp` подключаются к локальному демону.

| Флаг | Переменная | По умолчанию | Описание |
| --- | --- | --- | --- |
| `-token` | `VMFLOW_CONTROL_TOKEN` | _(нет)_ | Bearer token при включённой аутентификации. |

Для удалённого управления используйте SSH-туннель из раздела [Локальное управление](./api).

## Примечания

- Прежние раздельные бинарные файлы `relayd`, `relayctl` и `relaytui` всё ещё собираются для совместимости — это тонкие обёртки над теми же пакетами, читающие ту же переменную окружения `VMFLOW_CONTROL_TOKEN` — но в релизах предпочтение отдаётся единому бинарному файлу `vmflow`.
- Туннельные команды (`tunnel-server`, `tunnel-client`, `tunnel-ctl`) и команды сертификатов (`certs`, `certs-obtain`, `certs-review`) **не включены** в текущую сборку.
