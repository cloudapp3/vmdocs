# Движок перенаправления

Пакет `engine/` — это чистое ядро перенаправления. Ему принадлежат слушатели, циклы проксирования, сессии, счётчики и жизненный цикл правил — и больше ничего. Он не зависит от базы данных, плоскости управления или UI. Это сохраняет его встраиваемым.

## Протоколы

Каждое правило выбирает один протокол через поле `protocol`.

| Протокол | Поведение |
| --- | --- |
| `tcp` | Перенаправление портов TCP. |
| `udp` | Перенаправление портов UDP. |
| `tcp+udp` | Перенаправление и TCP, и UDP на одном порту из одного правила. |
| `http` | HTTP-прокси перенаправления (plain + `CONNECT`). **Отключено в текущей сборке.** |
| `https` | Перенаправление с завершением TLS и сертификатами на домен. **Отключено в текущей сборке.** |

`http` и `https` присутствуют в исходниках (`engine/https.go`, `engine/proxy.go`) вместе с пакетами ACME/certstore, но текущая сборка отклоняет их при валидации и не подключает управление сертификатами. Они зарезервированы для повторного включения в будущем.

## Управление на уровне правила

Каждое правило несёт независимые от протокола элементы управления временем выполнения:

- **`speed_limit`** — двустороннее ограничение скорости на соединение (token bucket), в байтах/сек. `0` означает отсутствие ограничений. Это намеренно простое, посессионное ограничение; это не общий разделяемый бакет полосы.
- **`max_conn`** — максимум одновременных соединений. При достижении лимита новые соединения немедленно закрываются. `0` означает отсутствие ограничений.
- **`idle_timeout`** — тайм-аут простоя на соединение в секундах. `0` означает использование значения по умолчанию (5 минут / 300 с). Изменение перезапускает правило.
- **`enabled`** — отключённое правило остаётся в конфигурации (и в снимках), но не запускается.

## Счётчики трафика

Каждое работающее правило сообщает счётчики в реальном времени:

- текущее количество соединений
- всего байт отправлено
- всего байт принято

Счётчики хранятся только в памяти. Для UDP количество соединений — это приблизительная сессионная оценка. Читайте их через [`GET /v1/stats`](./api#get-v1-stats) или через `vmflow ctl stats` / интерфейсы TUI.

## Эквивалентность правил

Правила несут как поля времени выполнения (протокол, адреса, порты, лимиты, `idle_timeout`), так и метаданные (`remark`, `revision`, временные метки). Движок сравнивает только поля времени выполнения при сопоставлении нового снимка с активным состоянием, поэтому косметические правки не вызывают лишних перезапусков — но изменение `idle_timeout` перезапуск всё же вызывает. См. [Правила и жизненный цикл](./rules).
