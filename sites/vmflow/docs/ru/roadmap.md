# Дорожная карта

vmflow находится на уровне практичного MVP стиля v0.1. Путь перенаправления, жизненный цикл правил, локальная плоскость управления, базовая наблюдаемость и API встраивания уже готовы.

## v0.1 — базовая линия

- [x] Перенаправление TCP
- [x] Перенаправление UDP
- [x] `ApplySnapshot`
- [x] демон + CLI
- [x] локальное управление через CLI/TUI
- [x] конфигурация YAML

## v0.2 — в работе

- [x] метрики Prometheus
- [x] улучшенное структурированное журналирование
- [x] аутентификация управления
- [x] предпроверка правил
- [ ] аккуратное освобождение соединений (graceful drain)
- [ ] ручная проверка на Windows / macOS

## v0.3 — запланировано

- [ ] общий бакет пропускной способности для каждого правила
- [ ] API подписки на события
- [ ] улучшения горячей перезагрузки конфигурации
- [x] нативная служба запуска при загрузке (systemd / launchd / Windows Service через `vmflow service install`)
- [ ] официальный образ Docker / примеры

## Зарезервировано на будущее

Перенаправление HTTP/HTTPS и управление ACME/сертификатами реализованы в исходном коде (`engine/https.go`, `engine/proxy.go`, `acme/`, `certstore/`, `certreview/`), но отключены в текущей сборке. Обход NAT (`tunnel/`) также сохранён, но не подключён. Оба — кандидаты на будущий релиз, когда L4-поверхность стабилизируется.
