# Правила и жизненный цикл

*Правило* — это единица, которой управляет vmflow: одна запись перенаправления `{protocol, listen, target}` с лимитами и метаданными. У правил есть жизненный цикл, которым можно управлять по одному или как единым снимком желаемого состояния.

## Стабильная идентичность

У каждого правила есть `rule_id`, который должен быть уникален в рамках конфигурации/снимка. По этому ID vmflow сопоставляет входящее правило с работающим, чтобы определить, что изменилось. Сохраняйте ID стабильными между перезагрузками — именно это делает возможным сравнение.

## Операции над одним правилом

| Операция | Эффект |
| --- | --- |
| `StartRule` | Запустить одно правило (оно не должно уже работать). |
| `RestartRule` | Остановить и запустить работающее правило, подхватив новые поля. |
| `StopRule` | Остановить работающее правило; сохранить его конфигурацию. |
| `RemoveRule` | Остановить правило и удалить его из активного набора. |

Они подходят для целевых локальных операций. Когда у вас есть полное желаемое состояние, предпочтительнее применение снимка.

## Применение снимка

`ApplySnapshot(rules, opts)` принимает полный желаемый набор правил и сверяет его с работающими. С `ReplaceAll` любое работающее правило, отсутствующее в снимке, останавливается.

Для каждого правила применение выдаёт одно действие:

| Действие | Значение |
| --- | --- |
| `started` | Правило есть в снимке, но не было запущено. |
| `restarted` | Правило работало, но его поля времени выполнения изменились. |
| `stopped` | Правило работало, но отсутствует в снимке (с `ReplaceAll`). |
| `removed` | Правило было остановлено и удалено. |
| `unchanged` | Правило работало, и его поля времени выполнения не изменились. |
| `failed` | Не удалось запустить правило (например, порт недоступен). |

…и сводку: `Applied`, `Stopped`, `Failed`, `Total`.

::: tip Что считается «изменением»?
Сравниваются только поля времени выполнения (протокол, адрес/порт прослушивания, адрес/порт назначения, ограничение скорости, максимум соединений, `idle_timeout`, enabled). Правка `remark` или временных меток **не** перезапускает правило.
:::

## Перезагрузка

`POST /v1/reload` перезагружает файл конфигурации и запускает `ApplySnapshot` с `ReplaceAll = true`. Сначала выполняется [предпроверка](./precheck); при любой ошибке перезагрузка отклоняется, а работающие правила остаются нетронутыми.

## Запросы состояния

| Метод | Возвращает |
| --- | --- |
| `RunningRules` / `RunningCount` | Текущие работающие правила / их количество. |
| `Snapshot(id)` | Живое состояние одного правила. |
| `SnapshotAll()` | Живое состояние всех работающих правил. |
| `StopAll()` | Остановить всё (например, при завершении работы). |

HTTP-эквиваленты (`GET /v1/rules`, `GET /v1/stats`) — см. [HTTP API](./api).
