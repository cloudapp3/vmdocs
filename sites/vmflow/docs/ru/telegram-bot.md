# Telegram-бот

vmflow может запускать опционального Telegram-бота для лёгкого чат-управления инспекцией и контролем — удобно, когда вы работаете с телефона и не хотите подключаться по SSH для быстрой проверки.

## Включение

Бот запускается только когда заданы **оба** поля:

```yaml
bot_token: <your-telegram-bot-token>
bot_chat: <chat-id>
```

`bot_token` — это токен от [@BotFather](https://t.me/BotFather); `bot_chat` — числовой ID чата, в котором боту разрешено отвечать. Если `bot_token` не задан, бот просто не запускается.

## Авторизация

Бот отвечает только в чате `bot_chat`. `bot_control_token` со значением admin token из `auth.tokens` включает запись; без него бот работает только на чтение. Запросы обрабатываются внутри процесса.

## Команды

| Команда | Эффект |
| --- | --- |
| `/start` | Показать текст справки. |
| `/status` | Количество работающих правил, итог по соединениям и итоги отправки/приёма. |
| `/rules` | Список работающих правил в таблице (имя, протокол, listen→target, соединения, отправлено/принято). |
| `/detail <id>` | Показать конфигурацию и живой трафик одного правила. |
| `/reload` | После подтверждения перечитать конфигурацию с диска. |
| `/stop <id>` | Остановить одно правило по ID (запрашивает подтверждение). |
| `/start_rule <id>` | После подтверждения включить и запустить правило. |

Разрушительные действия запрашивают подтверждение через inline-кнопки до того, как вступят в силу.

::: tip
Команды записи Bot используют ту же предпроверку, транзакционное применение, откат и оптимистичную блокировку, что и TUI. При конфликте предлагается повторить операцию.
:::

## Примечания

- Интеграция опциональна; основными средствами управления остаются CLI/TUI.
- Бот работает в том же процессе, что и демон, поэтому перезапуск демона перезапускает и бота.
