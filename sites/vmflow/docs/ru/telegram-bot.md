# Telegram-бот

vmflow может запускать опционального Telegram-бота для лёгкого чат-управления инспекцией и контролем — удобно, когда вы работаете с телефона и не хотите подключаться по SSH для быстрой проверки.

## Включение

Бот запускается только когда заданы **оба** поля:

```yaml
bot_token: <your-telegram-bot-token>
bot_chat: <chat-id>
```

`bot_token` — это токен от [@BotFather](https://t.me/BotFather); `bot_chat` — числовой ID чата, в котором боту разрешено отвечать. Если `bot_token` не задан, бот просто не запускается.

## Авторизация

Бот привязан к единственному чату, заданному в `bot_chat` — он игнорирует сообщения и callback-запросы из любых других чатов. Учтите, что бот обращается к внутрипроцессному движку **напрямую**, а не через control API, поэтому аутентификация control API для него не имеет значения. Раскрытие control API вне loopback всё ещё подчиняется собственным правилам (см. [Развёртывание](./deployment)).

## Команды

| Команда | Эффект |
| --- | --- |
| `/start` | Показать текст справки. |
| `/status` | Количество работающих правил, итог по соединениям и итоги отправки/приёма. |
| `/rules` | Список работающих правил в таблице (имя, протокол, listen→target, соединения, отправлено/принято). |
| `/detail <id>` | Показать конфигурацию и живой трафик одного правила. |
| `/reload` | Предлагает перезагрузку; при подтверждении **останавливает все правила** (см. примечание ниже). |
| `/stop <id>` | Остановить одно правило по ID (запрашивает подтверждение). |
| `/start_rule <id>` | Предлагает запустить правило (см. примечание ниже). |

Разрушительные действия запрашивают подтверждение через inline-кнопки до того, как вступят в силу.

::: warning Ограничения `/reload` и `/start_rule`
- `/reload` **не** перечитывает конфигурацию. При подтверждении он вызывает `StopAll()` и отвечает: _"✅ All rules stopped. Use control API /v1/reload for full config reload."_ Чтобы реально применить изменения конфигурации, выполните `vmflow ctl reload` (или `POST /v1/reload`) против control API.
- `/start_rule` — заглушка: при подтверждении он отвечает подсказкой использовать control API `/v1/reload`, поскольку повторное применение одного правила требует полной конфигурации правила, которой у бота нет.

`/stop <id>`, напротив, работает полностью — он немедленно останавливает указанное правило.
:::

## Примечания

- Эта интеграция опциональна и работает по принципу best-effort; она не заменяет control API для продакшен-перезагрузок.
- Бот работает в том же процессе, что и демон, поэтому перезапуск демона перезапускает и бота.
