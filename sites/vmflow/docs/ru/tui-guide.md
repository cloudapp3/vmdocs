# Панель TUI

vmflow поставляется с терминальным интерфейсом для просмотра и управления настроенными правилами и оперативными счётчиками трафика локального демона.

## Запуск

```bash
vmflow tui
```

Если аутентификация включена, передайте токен доступа:

```bash
vmflow tui -token <token>
# or
VMFLOW_CONTROL_TOKEN=<token> vmflow tui
```

## Режимы

Нажмите <kbd>Tab</kbd> для циклического переключения между режимами:

| Режим | Показывает |
| --- | --- |
| **Dashboard** | Общее состояние здоровья, количество работающих правил, время работы (uptime). |
| **Rules** | Настроенные правила, включая отключённые, оперативные счётчики, подготовленные изменения и сводка доступа `OPEN` / `ALLOW n` / `DENY n` на широких терминалах. |
| **Detail** | Настройки выбранного правила, записи IP-адресов источника, трафик и накопительный счётчик `IP Denied`. |

## Управление правилами

С аутентифицированным токеном `admin` можно создавать, редактировать, копировать, включать, отключать и удалять правила. Токены viewer и сеансы без аутентификации доступны только для чтения. В режиме Rules используйте <kbd>n</kbd>/<kbd>e</kbd>/<kbd>c</kbd> для создания, редактирования и копирования; <kbd>Space</kbd> для переключения; <kbd>d</kbd> для удаления; <kbd>P</kbd> для предпроверки и <kbd>A</kbd> для применения проверенного черновика.

В редакторе поле `Source IP mode` принимает `OFF`, `ALLOWLIST` или `DENYLIST`. В `Source IPs / CIDRs` вводите литеральные адреса IPv4/IPv6 и CIDR через запятую. Перед применением предпроверка должна завершиться успешно; существующий механизм revision/ETag не позволяет устаревшему редактору перезаписать более новую конфигурацию.

## Когда использовать

TUI — самый быстрый способ ответить на вопрос «что vmflow делает прямо сейчас?» без сбора метрик. Для долгосрочной истории направьте Prometheus на `/metrics` — TUI показывает только текущее состояние из памяти.
