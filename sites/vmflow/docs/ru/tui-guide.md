# Панель TUI

vmflow поставляется с терминальным интерфейсом для инспекции работающего демона. Он читает данные из локального control API, поэтому показывает живое состояние правил и счётчики трафика.

## Запуск

```bash
vmflow tui
```

Укажите нестандартный адрес управления или передайте токен:

```bash
vmflow tui -addr http://127.0.0.1:19090 -token <token>
# or
VMFLOW_CONTROL_TOKEN=<token> vmflow tui
```

TUI принимает те же клиентские флаги, что и `ctl`, включая флаги TLS/mTLS (`-tls-ca-file`, `-tls-client-cert`, `-tls-client-key`, `-tls-skip-verify`) и `-H` / `--header` для пользовательских заголовков запроса.

## Режимы

Нажмите <kbd>Tab</kbd> для циклического переключения между режимами:

| Режим | Показывает |
| --- | --- |
| **Dashboard** | Общее состояние здоровья, количество работающих правил, время работы (uptime). |
| **Rules** | Список работающих правил с живыми счётчиками; поддерживает фильтрацию правил по имени. |
| **Detail** | Подробности о выбранном правиле. |

## Когда использовать

TUI — самый быстрый способ ответить на вопрос «что vmflow делает прямо сейчас?» без сбора метрик. Для долгосрочной истории направьте Prometheus на `/metrics` — TUI показывает только текущее состояние из памяти.
