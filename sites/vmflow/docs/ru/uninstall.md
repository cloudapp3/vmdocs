# vmflow uninstall

```bash
vmflow uninstall [--dry-run]
```

Псевдонимы: `vmflow remove`, `vmflow rm`.

Выполняет полное удаление установки vmflow. Процесс идёт по схеме **план → подтверждение → выполнение**:

1. **План** — проверяет систему и выводит всё, что будет удалено.
2. **Подтверждение** — запрашивает `[y/N]` (пропускается при `--dry-run` или когда удалять нечего).
3. **Выполнение** — удаляет элементы по порядку, допуская уже отсутствующие пути, так что команда идемпотентна.

## Флаги

| Флаг | По умолчанию | Описание |
| --- | --- | --- |
| `--dry-run` | `false` | Выводит план удаления, ничего не удаляя. |

## Что удаляется

Элементы удаляются в следующем порядке (сначала служба, работающий бинарный файл — последним, поэтому демон, ещё находящийся под супервизором, исчезает до удаления его исполняемого файла):

| Элемент | Примечания |
| --- | --- |
| Нативная служба | Останавливает и удаляет systemd unit / launchd plist / Windows Service. |
| Файл конфигурации | Конфигурация платформы по умолчанию (см. [`service`](./service)), если она есть. |
| Внешние сертификаты TLS / ACME | Файлы сертификатов и ключей вне хранилища vmflow сохраняются. |
| Каталоги журналов | например `/var/log/vmflow` (Linux/macOS), `C:\ProgramData\vmflow\logs` (Windows). |
| Кэш самообновления | Каталог кэша обновлений. |
| Бинарный файл vmflow | Работающий исполняемый файл, удаляется последним. |

::: warning Установки через пакетный менеджер
Если бинарный файл принадлежит пакетному менеджеру (`dpkg` / `rpm`), `uninstall` выводит предупреждение и рекомендует вместо этого `apt remove` / `yum remove`, поскольку прямое удаление файла оставляет базу пакетов несинхронизированной. При подтверждении он всё равно продолжит.
:::

Защищённые пути (системные корни, ваш домашний каталог) никогда не удаляются.

## Примеры

```bash
# preview only
vmflow uninstall --dry-run

# full teardown (will prompt to confirm)
sudo vmflow uninstall
```
