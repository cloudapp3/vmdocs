# 转发引擎

`engine/` 包是纯转发核心。它拥有监听器、代理循环、会话、计数器和规则生命周期——除此之外什么都没有。它不依赖数据库、控制面或 UI。这正是它能被嵌入的原因。

## 协议

每条规则通过 `protocol` 字段选择一种协议。

| 协议 | 行为 |
| --- | --- |
| `tcp` | TCP 端口转发。 |
| `udp` | UDP 端口转发。 |
| `tcp+udp` | 用单条规则在同一端口上同时转发 TCP 和 UDP。 |
| `http` | HTTP 转发代理（明文 + `CONNECT`）。**在当前构建中已禁用。** |
| `https` | 带每域名证书的 TLS 终结转发。**在当前构建中已禁用。** |

`http` 和 `https` 存在于源码中（`engine/https.go`、`engine/proxy.go`），与 ACME/certstore 包并列，但当前构建在校验阶段就会拒绝它们，并且不会接通证书管理。它们留待日后重新启用。

## 每规则控制项

每条规则都携带与协议无关的运行时控制项：

- **`speed_limit`**——每连接的双向速率限制（令牌桶），单位为 bytes/sec。`0` 表示不限制。这是有意设计的简单、按会话生效的限制；它不是一个共享的全局带宽桶。
- **`max_conn`**——并发连接数上限。当达到上限时，新连接会被立即关闭。`0` 表示不限制。
- **`idle_timeout`**——每连接的空闲超时，单位为秒。`0` 表示使用默认值（5 分钟 / 300 秒）。修改此项会重启该规则。
- **`source_ip_mode` / `source_ips`**——`off`、IPv4/IPv6 白名单或黑名单。条目可以是字面 IP 或 CIDR，每条规则最多 256 项。
- **`enabled`**——被禁用的规则会保留在配置中（以及快照中），但不会被启动。

## 来源 IP 准入

TCP 在 `Accept` 后、占用连接槽和连接目标端之前检查 socket 对端。UDP 在创建会话或占用任一 UDP 会话配额之前检查每个来源数据报。因此，被拒绝的流量不会消耗转发容量。

有效来源策略发生变化时，规则会重启并关闭已有 TCP 连接和 UDP 会话；仅调整等价 CIDR 的顺序不会重启。策略只使用 socket 对端地址：vmflow 不信任 HTTP 转发头或 PROXY protocol 元数据，上游 NAT 或四层代理可能隐藏原始客户端地址。

## 流量计数器

每条运行中的规则都会上报实时计数器：

- 当前连接数
- 上传字节总数
- 下载字节总数
- 来源 IP 拒绝总数（TCP 连接尝试或 UDP 数据报）

可通过 `vmflow ctl stats` 或 TUI 查看计数器。来源策略拒绝数也会导出为 `vmflow_rule_source_ip_denied_total`；启用统计持久化后，该总数会跨守护进程重启保留。UDP 的连接数是近似会话数。

## 规则等价性

规则同时携带运行时字段（protocol、地址、端口、限制、`idle_timeout`、来源 IP 策略）和元数据（`remark`、`revision`、时间戳）。引擎在将新快照与运行状态进行 diff 时只比较运行时字段，因此外观上的编辑不会导致不必要的重启——但修改 `idle_timeout` 或有效来源 IP 策略会触发重启。参见[规则与生命周期](./rules)。
