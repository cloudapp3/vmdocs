# `vminfo net`

Запускайте разовую сетевую диагностику из того же бинарника, который вы уже
используете для метрик хоста. Каждая подкоманда по умолчанию выводит читаемый
результат и поддерживает `--json` для скриптов.

## Подкоманды

| Действие | Что делает |
| --- | --- |
| `net dns` | Разрешает домен в адреса |
| `net port` | Проверяет TCP-соединение и задержку до `host:port` |
| `net ping` | Зондирует хост, измеряя RTT через TCP-подключение или ICMP |
| `net ip` | Определяет публичный IP, а также ASN и гео-данные |

## `net dns`

```bash
vminfo net dns example.com
vminfo net dns example.com --server 1.1.1.1
vminfo net dns example.com --server 1.1.1.1 --json
```

| Флаг | Описание |
| --- | --- |
| позиционный домен | Домен для разрешения (ровно один) |
| `--server` | DNS-сервер в виде `host` или `host:port`; пусто = системный по умолчанию |
| `--json` | Вывести результат в виде JSON |

## `net port`

```bash
vminfo net port example.com 443
vminfo net port example.com 443 --timeout 3s --json
```

| Флаг | Описание |
| --- | --- |
| `host` / `port` | Целевые хост и порт (1–65535) |
| `--timeout` | Таймаут подключения, по умолчанию `2s` |
| `--json` | Вывести результат в виде JSON |

## `net ping`

```bash
vminfo net ping example.com                       # TCP ping, порт 80
vminfo net ping example.com --tcp-port 443        # TCP ping на 443
vminfo net ping example.com --mode icmp           # настоящий ICMP ping (нужны привилегии)
vminfo net ping example.com --count 6 --json
```

| Флаг | Описание |
| --- | --- |
| позиционный хост | Хост для зондирования (ровно один) |
| `--mode` | `tcp` (по умолчанию) или `icmp` |
| `--count` | Число проб, по умолчанию `4` |
| `--timeout` | Таймаут на пробу, по умолчанию `1s` |
| `--tcp-port` | Целевой TCP-порт, по умолчанию `80` (только режим tcp) |
| `--json` | Вывести результат в виде JSON |

::: tip TCP против ICMP
Режим `tcp` измеряет RTT через TCP-подключение — он кроссплатформенный и не
требует привилегий, поэтому работает везде. Режим `icmp` отправляет настоящие
пакеты ICMP Echo через `golang.org/x/net`; на Linux требуется
`net.ipv4.ping_group_range` для предоставления непривилегированного UDP
ICMP-сокета, а на Windows он не поддерживается. Если ICMP недоступен,
переключитесь на `--mode tcp`.
:::

## `net ip`

```bash
vminfo net ip                       # ваш собственный публичный IP + ASN / гео
vminfo net ip 8.8.8.8               # поиск конкретного IP
vminfo net ip --json
```

| Флаг | Описание |
| --- | --- |
| позиционный ip | Необязательный IP для поиска; если опущен — ваш собственный публичный IP |
| `--server` | Базовый URL сервиса поиска, по умолчанию `https://ip.bestcheapvps.org` |
| `--json` | Вывести результат в виде JSON |

::: warning Исходящий запрос
`net ip` выполняет явный, инициированный пользователем запрос к стороннему
сервису поиска (`ip.bestcheapvps.org` по умолчанию), чтобы получить ASN,
гео-данные и флаги риска. Это описано в `--help` и в выводе команды. Запрос
никогда не выполняется автоматически — только когда вы его запрашиваете.
:::

## Примечания

- Читаемый вывод локализован; вывод JSON стабилен и не зависит от языка.
- JSON-результаты возвращают тайминги в `elapsed_ms` и ошибки в поле `error`.
- Эти диагностики также доступны из веб-дашборда через
  [`POST /api/v1/net/diag`](/ru/api#post-api-v1-net-diag).

## Пример

```bash
vminfo net ping example.com --tcp-port 443 --count 4 --json
```
