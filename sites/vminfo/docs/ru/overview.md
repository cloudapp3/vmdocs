# Обзор

vminfo — кроссплатформенный набор инструментов для получения информации о рантайме хоста, предназначенный для быстрой локальной диагностики.

## Что вы получаете

- **TUI** — полноэкранный терминальный дашборд
- **JSON-вывод** — удобные для скриптов снимки и потоки watch
- **Веб-дашборд** — легковесный браузерный UI с REST- и WebSocket-эндпоинтами
- **Библиотека Go** — импортируемый сбор метрик и API встраиваемого TUI

## Собираемые метрики

vminfo собирает:

- использование CPU, загрузку по ядрам, частоту CPU
- использование памяти и swap
- использование диска и дисковый ввод-вывод
- итоги по сети, скорости, количество соединений TCP/UDP
- распределение состояний TCP (`ESTABLISHED` / `TIME_WAIT` / `SYN_RECV` / …) и использование conntrack (Linux)
- трафик, ошибки и потери по интерфейсам
- списки процессов и метаданные процессов
- показатели температуры
- uptime и метаданные хоста

## Публичные типы Go

- `StaticInfo`
- `RuntimeStats`
- `Snapshot`
- `ProcessInfo`
- `AppMetadata`

## Основные точки входа

- `vminfo`
- `vminfo info`
- `vminfo summary`
- `vminfo watch`
- `vminfo --web`
- `vminfo ps`
- `vminfo kill <pid>`
- `vminfo net dns | port | ping | ip`
- `vminfo update`

## Когда подходит

vminfo хорошо подходит, когда вам нужно:

- один бинарник для быстрого осмотра хоста
- читаемый терминальный UI для интерактивной работы
- JSON для скриптов, CI или автоматизации
- веб-дашборд без развёртывания стека мониторинга
- библиотека Go, встраиваемая в другой CLI
